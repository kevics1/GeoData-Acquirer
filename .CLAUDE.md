# GeoData-Acquirer

Graduation project. LLM-based intelligent geographic data acquisition system. Supervisor: 洪松教授.

## Key facts
- Python 3.12, no virtualenv (system Python at `D:\Python312`)
- The `thefuck` CLI tool is broken (Python 3.12 incompatibility) — ignore its traceback noise on every command
- Windows 11, shell is bash; use forward slashes in paths
- Windows console prints garbled Chinese (GBK encoding issue) — output files use UTF-8 and are unaffected

## Commands
- Run: `python cli.py "<natural language query>"`
- List sources: `python cli.py --list-sources`
- Force source: `python cli.py "<query>" --source <name>`

## Environment
- `.env` at `config/.env` — loaded by `cli.py` via `python-dotenv`
- LLM: ModelScope DeepSeek-V3.2 (`GEODATA_LLM_API_KEY`, `GEODATA_LLM_BASE_URL`, `GEODATA_LLM_MODEL`)
- DB: PostgreSQL+PostGIS (`GEODATA_DB_HOST/PORT/USER/PASSWORD/NAME=Administrator`)
- DB tables: `province`, `city`, `county`, `village` with code fields `省级码`/`地级码`/`县级码`/`area_code`

## Architecture
```
cli.py → src/main.py (GeoDataAcquirer)
  ├── src/llm_parser.py   — LangChain + DeepSeek, extracts region/data_type/temporal/resolution
  ├── src/kg_query.py      — 37,905-node admin KG (name→code mapping)
  ├── src/matcher.py       — weighted scoring: quality(0.4) + auth(0.2) + coverage(0.2) + resolution(0.2)
  ├── src/downloader.py    — HTTP with 3-retry backoff + tqdm progress
  └── src/datasources/
      ├── base.py          — GeoDataSource ABC: can_handle/search/download/get_metadata
      ├── registry.py      — DataSourceRegistry singleton, auto-registration
      ├── admin_boundary.py — wraps kg_query + db_connector (data_type="boundary")
      ├── osm.py           — Overpass API (road/building/poi/landuse/waterway)
      └── open_meteo.py    — free weather API (weather/climate/air_quality), uses Wuhan lat/lon default
```

## Data source routing
- `data_type="boundary"` → admin_boundary (PostGIS, China only)
- `data_type in [road, building, poi, landuse, waterway]` → OSM (Overpass API, global)
- `data_type in [weather, climate, air_quality]` → Open-Meteo (REST API, global, no auth)
- Unknown data_type → NoSourceFoundError
